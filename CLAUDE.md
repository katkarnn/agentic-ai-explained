# CLAUDE.md — "How Agentic AI Actually Works" LinkedIn Carousel

Context and working instructions for editing this slide deck from Claude Code.

## What this is

A **square (1:1) 36-slide LinkedIn carousel** titled *"How Agentic AI Actually Works."*
It teaches agentic AI from zero — written by a developer, for first-time readers, in
school-kid-simple language with terminal/code examples and diagrams on almost every slide.

It is generated programmatically with **PptxGenJS** (Node). The deck is **regenerated from
source**, not hand-edited in PowerPoint. Workflow: edit `build.js` → run `render.ps1` →
visually QA the rendered JPGs → repeat.

## Files

| `render-short.ps1` | Pipeline for the short cut: `make-short.js` → `build-short.js` → `Agentic-AI-Explained-Short.pdf` + `short-slide-01..20.jpg`. |
| `Agentic-AI-Explained-Short.pdf` / `short-slide-*.jpg` | Generated 20-slide cut. JPGs = a **multi-image LinkedIn post** (LinkedIn caps multi-image at 20). |

## Build + QA loop

```powershell
powershell -ExecutionPolicy Bypass -File render.ps1
```

That builds the `.pptx`, converts to PDF, and writes `slide-01.jpg … slide-36.jpg` at 120 dpi.
Then **open every JPG and read each text box** — the #1 defect is text overflowing its box or a
stacked list running past the bottom margin. (soffice prints a harmless "Could not find platform
independent libraries" warning to stderr; the script ignores it — the PDF is still produced.)

**To export for LinkedIn:** upload `Agentic-AI-Explained.pdf` as a **document post**. It's already
a valid carousel.

## Design system — HYBRID theme (do not drift from this)

**The look:** light, high-contrast slides (most readable in sunlight) with embedded **dark
terminal/code panels** for the developer feel. Dark slides are used only as section dividers and
keystone moments. There is **no PowerPoint setting that adapts to ambient light** — readability in
both sunny and dark conditions comes entirely from this high-contrast palette.

**Layout:** custom square `10in × 10in` (`pres.defineLayout({ name: "SQ", width: 10, height: 10 })`).
All coordinates in inches. Keep ≥ 0.5" margins.

**Palette** (hex, no `#`):

| Token | Hex | Use |
|-------|-----|-----|
| `DARK` | `082A30` | dark teal — divider / hero / keystone backgrounds |
| `DARK2` | `0C3942` | cards & decorative circles on dark |
| `BG` | `FFFFFF` | light slide background (most slides) |
| `BGSOFT` | `F3F9FA` | alternate light background |
| `CARD` | `F1F8F9` | subtle card tint on light |
| `INK` | `0B2329` | body text on light (near-black, high contrast) |
| `HEAD` | `082A30` | headers on light |
| `TEAL` | `027A8A` | primary accent (fills, badges) |
| `TEALTX` | `045E6B` | darker teal for accent **text** on light (contrast-safe) |
| `MINT` | `06D6A0` | vibrant accent — fills/badges, or text **on dark only** |
| `MINTTX` | `0A7A5C` | dark mint for mint-toned text on light |
| `AMBER` / `AMBERTX` | `E8893A` / `B45F16` | warnings / "gotcha" / pitfalls |
| `MUTED` | `4B666E` | captions / secondary text on light |
| `LINE` | `D7E6E9` | hairline card borders |

**Terminal-panel palette** (dark): `TBG 0A1E22` bg · `TBORDER 16434C` · `TGREEN 5EEAA8` (prompt/
success) · `TCYAN 6FD3E6` (keywords/tools) · `TYEL F2C14E` (strings/values) · `TCMT 7FA0A8`
(comments) · `TWHITE E8F4F6` (default) · `TRED E76F51` (window dot/errors).

> Contrast rule: never put `MINT` or plain `TEAL` as small text on white — they fail contrast.
> Use `TEALTX` / `MINTTX` for accent text on light; `MINT` is fine as text on dark.

**Fonts** (all QA-safe / metric-stable):
- `HF = "Cambria"` — serif, titles & big headers.
- `BF = "Calibri"` — sans, all body, kickers, captions, bullets.
- `MONO = "Consolas"` — terminal panels, code, kickers on dark, page numbers.

**Motif:** white icons (react-icons/fa) inside filled circular **badges** (`badge()`), rounded
**cards** with a soft shadow, and dark **terminal panels** (`term()`) with a 3-dot title bar.
Diagrams use rounded-rect **nodes** (`node()`) connected by **arrows** (`arrow()`, real arrowheads).
**Never** add accent stripes, edge bars, or title underlines — they read as AI-generated filler.

**Structure:** the deck is 6 parts, each opened by a dark `divider()` slide. Dark slides: 1 (title),
4/8/13/21/30 (dividers), 10 (ReAct), 22 (Workflow vs Agent keystone), 36 (closing). Everything else
is light. Page numbers (`n / 36`) are added automatically at the end from the `slides[]` array — you
never hard-code the total.

### Reusable helpers in `build.js`

- `newSlide(bg, onDark)` — adds a slide, sets bg, registers it for auto page-numbering. Use this, not `pres.addSlide()`.
- `badge(slide, x, y, d, fill, icon)` — filled circle (diameter `d`) with a white icon centered.
- `card(slide, x, y, w, h, fill)` — rounded rect + hairline border + shadow.
- `kicker / title / lead / note` — the standard top-of-slide text rhythm (kicker `y0.52` → title `y0.95` → lead `y~2.0` → note near bottom).
- `chip(...)` / `node(...)` — small labelled pills / diagram boxes.
- `arrow(slide, x1,y1,x2,y2,color,width)` — line with a real triangle arrowhead (use `TEAL`/`TEALTX` on light, `MINT`/`MINTTX` on dark; **never white on white**).
- `term(slide, x, y, w, h, filename, lines, opt)` — dark terminal panel. `lines` is an array of rows; each row is an array of runs `{ t: text, c: color, b: bold }`. Last run of each row auto-breaks. `opt.fs` font size, `opt.ls` line spacing.
- `divider(part, title, subtitle, icon)` — dark section-divider slide.
- `patternSlide(cfg)` — shared layout for the 5 pattern zoom-ins (slides 24–28): kicker, title, lead, a `cfg.diagram(s)` callback, a terminal example, and a closing note.
- `mkShadow()` — **returns a fresh shadow object each call.** Always call it inline; never store and reuse one (PptxGenJS mutates option objects in place).
- Icons are pre-rendered into the `I` map (white PNGs) from the `need = {…}` object near the top. Add an icon by adding to `need`.

## Common edit recipes

- **Change wording:** find the slide's array/`cfg` and edit the strings. Rebuild + QA.
- **Edit a terminal example:** find its `term(...)` call; rows are `[{t,c}, …]`. Keep lines short (< ~55 mono chars) and watch the line count against the panel height.
- **Add a comparison bullet (slides 12/18/22/29):** add a string to that card's bullet array; watch height — drop `fontSize` or raise the card `h` if it overflows.
- **Add a pattern:** add a `patternSlide({...})` call with a `diagram(s)` callback.
- **Recolor:** change the palette constants at the top; everything references the tokens. Keep one accent dominant.
- **Swap an icon:** add a `react-icons/fa` name to `need = {}`, then reference `I.yourkey`.

## PptxGenJS pitfalls (these corrupt the file or render wrong)

1. **Hex colors never use `#`** → `color: "FF0000"` ✓, `"#FF0000"` ✗ (corrupts file). (The `icon()` helper *does* take `#`-prefixed CSS colors — that's SVG, not PptxGenJS.)
2. **Shape names:** use `pres.shapes.RECTANGLE`, `ROUNDED_RECTANGLE`, `OVAL`, `LINE`. `RECT` is **not** valid and throws "Missing/Invalid shape parameter".
3. **No 8-char hex / no opacity baked into color strings** → use `opacity` on shadows, `transparency` on fills.
4. **Never reuse a shadow (or any option) object** → call `mkShadow()` fresh each time.
5. **Bullets:** use `{ bullet: true }`, never a literal `"•"` (double bullets).
6. **Multi-line text:** use `breakLine: true` between runs.
7. **Arrows must contrast with their background** — `arrow()` is teal on light, mint on dark. A white arrow on a white slide is invisible.
8. **No decorative color bars / edge stripes / title underlines** — set a card apart with a `fill` tint or `shadow`, not a stripe.

## QA acceptance bar

After a change, run `render.ps1` and inspect the affected JPGs for: text overflow / cut-off,
stacked lists running past the bottom margin, overlapping elements, low-contrast text, uneven gaps,
and margins < 0.5". Fix real defects; don't chase sub-pixel nudges.
